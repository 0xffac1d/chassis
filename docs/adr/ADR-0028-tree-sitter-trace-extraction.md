# ADR-0028 — Tree-sitter trace extraction (optional)

## Context

ADR-0023 standardized a **line-oriented** `@claim` scanner. Tree-sitter can classify line comments without mistaking string literals for comments and can flag whole-file parse errors.

## Decision

- **Default remains regex/line scan** (`TraceExtractBackend::Regex`) so behavior is unchanged for `build_trace_graph` consumers.
- **`TraceExtractBackend::TreeSitter`** parses each file with the upstream Rust / TypeScript grammars, collects `line_comment` (Rust) or `//` `comment` nodes (TS/TSX), and only starts a claim block on those lines.
- **`CH-TRACE-PARSE-ERROR` (warning):** emitted once per file when the tree has error nodes or parse returns no tree.
- **Generated sources:** if the first non-shebang non-empty line contains `@generated` or `// @generated by`, the file is skipped in tree-sitter mode.
- **`violated.convention`:** `ADR-0028` for parse-error diagnostics; ADR-0023 remains authoritative for malformed-claim and site semantics.

## Consequences

- CI and `chassis trace` may opt into `--extractor tree-sitter`; parity tests require both backends to agree on claim sites for this repository.
- **Trace id:** `chassis.trace-tree-sitter-parity` records the parity gate for regex vs tree-sitter backends.
